Task Definition & Key Challenges

Multi-label classification: assign one or more ICD-10 codes to a short, unstructured physician note in Spanish or Catalan.

Mixed language & abbreviations: clinical phrases mix Spanish, Catalan and heavy abbreviations (HTA, ITU, IRC). No standard tokeniser handles this reliably.

Large, imbalanced label space: 4,059 unique codes. 47 % appear only once — a severe long-tail distribution where most classifiers will fail silently.

Multi-label mapping: one literal can correspond to up to 14 ICD-10 codes simultaneously, requiring a One-vs-Rest strategy.

Three Comparative Baselines — Survey §4.2 (Table 1)

Primary baseline Perotte et al. [32]

TF-IDF + FlatSVM

Word-level TF-IDF features fed into a One-vs-Rest Linear SVM. Each ICD code is treated as an independent binary classification problem. The literature's gold standard for this task.

Secondary baseline Marafino et al. [37]

Char N-gram + SVM

Character-level n-grams (3–5 chars) as features. Naturally captures abbreviations (HTA, ITU) and handles Spanish/Catalan shared roots without a curated medical dictionary.

Fast baseline Elyne et al. [16]

Bag-of-Words + Naive Bayes

BoW features with Complement Naive Bayes (suited for imbalanced data). Near-instant training. Sets the performance floor against which SVM complexity can be justified.

Known Limitations & Our Mitigation Plan

Computational Impracticality (Survey §4.2)

Training 4,059 separate SVM classifiers is prohibitive. Mitigation: April 8th baseline filters to the top-N most frequent codes to ensure each classifier has sufficient positive examples, then we scale up progressively.

Code Dependency Ignored (Survey §4.2)

Flat SVMs treat each code independently, missing co-occurrence patterns between diagnoses. Future work: deep learning models that capture these dependencies will be the next milestone after baselines are established.
